## Overview

The **Document Quality Control (QC) Workspace** is a developer-focused frontend toolkit designed to accelerate the creation of sophisticated document review interfaces. Built with a modern technology stack including React, TypeScript, Vite, Tailwind CSS, and Radix UI, it provides a robust library of reusable, accessible UI components and a scalable architecture.

This toolkit simplifies the process of building complex, interactive applications for reviewing and correcting data extracted from business documents.

## Getting Started

Follow these steps to get a local copy of the project up and running.

### Prerequisites

This project requires the following dependencies:

  * **Programming Language:** TypeScript
  * **Package Manager:** npm (or yarn/pnpm)

### Installation

Build the project from the source and install its dependencies:

1.  **Clone the repository:**

    ```bash
    git clone https://github.com/Sanganisathwik/document-qc
    ```


2.  **Navigate to the project directory:**

    ```bash
    cd document-qc
    ```

   

3.  **Install the dependencies:**
    Using `npm`:

    ```bash
    npm install
    ```

### Usage

Run the development server to start the project:

Using `npm`:

```bash
npm run dev
```

*(Note: The command is typically `npm run dev` for Vite projects, but check your `package.json` scripts. The example showed `npm start`)*

### Testing

This project uses a test framework for ensuring code quality. Run the test suite with:

Using `npm`:

```bash
npm test
```
